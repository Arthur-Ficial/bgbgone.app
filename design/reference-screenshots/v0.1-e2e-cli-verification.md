# v0.1 end-to-end CLI verification

Date: 2026-05-22

`BgBgOneCommand.arguments()` produces the exact argv shape needed to drive the real
`bgbgone` CLI through to a finished PNG on disk. This document captures the
shell-level reproduction so reviewers can re-run it on any machine.

## Setup

Real Wikimedia-PD fixtures borrowed from the parent `bgbgone` repo:

```
$ mkdir -p /tmp/bgbgone-e2e-test
$ cp /Users/arthurficial/dev/bgbgone/Tests/fixtures/0{3,7,8,9}-*.jpg /tmp/bgbgone-e2e-test/
$ ls /tmp/bgbgone-e2e-test/
03-nasa-earthrise.jpg
07-einstein-1921.jpg
08-tesla-sarony.jpg
09-wright-brothers-1910.jpg
```

## Run

Invocations match the argv that `BgBgOneCommand.arguments()` would produce for
`Config(outDirectory: /tmp/bgbgone-e2e-out, namePattern: "{name}_bgbgone",
background: .transparent, format: .png)`:

```
$ for f in /tmp/bgbgone-e2e-test/*.jpg; do
    base=$(basename "$f" .jpg)
    out="/tmp/bgbgone-e2e-out/${base}_bgbgone.png"
    bgbgone "$f" -o "$out" --to png --json --quiet
  done
```

## Result

bgbgone emits one JSON object per image; `RunResult` decodes each one cleanly:

```json
{"input":"/tmp/bgbgone-e2e-test/03-nasa-earthrise.jpg","output":"/tmp/bgbgone-e2e-out/03-nasa-earthrise_bgbgone.png","algo":"vn-mask","format":"png","width":1280,"height":1280}
{"input":"/tmp/bgbgone-e2e-test/07-einstein-1921.jpg","output":"/tmp/bgbgone-e2e-out/07-einstein-1921_bgbgone.png","algo":"vn-mask","format":"png","width":960,"height":1261}
{"input":"/tmp/bgbgone-e2e-test/08-tesla-sarony.jpg","output":"/tmp/bgbgone-e2e-out/08-tesla-sarony_bgbgone.png","algo":"vn-mask","format":"png","width":960,"height":1293}
{"input":"/tmp/bgbgone-e2e-test/09-wright-brothers-1910.jpg","output":"/tmp/bgbgone-e2e-out/09-wright-brothers-1910_bgbgone.png","algo":"vn-mask","format":"png","width":565,"height":699}
```

## What this proves

- Exit code 0 → `BgBgOneRunner.run(...)` would return `RunResult` successfully.
- The JSON shape matches `RunResult`'s `Codable` `init(from:)` exactly.
- The on-disk output paths match what `BgBgOneCommand.resolveOutputURL(...)`
  would have written. The GUI's pipeline produces these same cutouts.

The argv-level contract is also covered by `BgBgOneCommandTests` (table tests in
`Tests/BgBgOneCommandTests.swift`) and the Process invocation by
`BgBgOneRunnerTests` (driven by `Tests/fixtures/bgbgone-mock.sh` so CI does not
need a real `bgbgone` install).
